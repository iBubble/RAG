# 技术决策记录 (ADR)

## ADR-001: 视频与音频多模态文件解析技术选型

### 背景
系统需要支持对视频（mp4/mov）和音频（mp3/wav）文件进行语义检索。这就需要先将音频或视频中的语音内容提取为纯文本，再将其送入向量化管道。

### 备选方案对比

#### 方案 1: 本地 ffmpeg + transformers ASR Pipeline (Whisper-Base)
- **原理**：调用系统级 `ffmpeg` 命令进行格式标准化（转为 16kHz 单声道 wav），再通过 PyTorch/transformers 的 ASR Pipeline 运行 `openai/whisper-base` 本地模型识别语音。
- **优点**：100% 本地化，安全合规，对 CPU/GPU 性能开销适中，识别率高，扩展性强。
- **缺点**：容器内需要安装 `ffmpeg`，以及首次运行需在线下载 whisper 权重（约 74MB）。

#### 方案 2: Ollama 视觉模型分帧结合分析
- **原理**：将视频分帧送至 `qwen2.5vl:7b` 视觉模型进行关键帧汇总描述。
- **优点**：可以捕捉图像细节。
- **缺点**：显存开销极大，容易造成 GPU 击穿，且完全丢失了当事人录音的语音对话内容。

#### 方案 3: 云端第三方 Whisper API 接入
- **原理**：直接将音视频文件上传到外部云端接口进行转写。
- **优点**：本地开发无需配置任何 ASR 模型和 ffmpeg。
- **缺点**：不符合本地安全及企业图纸/证据保密隔离原则，需要外部 Key。

### 技术决策
**选择【方案 1】**。由于该系统是私有化、局域网安全部署的 RAG 知识库，且运行在 MacBook Pro / Mac Studio 等高性能设备上，本地 Whisper-Base 能在保障隐私的前提下提供毫秒级的极速语音识别响应。
