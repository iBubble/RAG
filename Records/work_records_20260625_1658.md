# 2026-06-25 16:58 页面宽度、表格外框线与页边距及宋体高保真还原重构记录

## 修改背景与目的
根据 Master 的反馈，在前台与后台的 AI 表格编辑页面中：
1. 宽度需要加宽，同时要能直观体现出“页边距”与纸张边缘的关系，像 Word 中排版一样；
2. 表格外框线在渲染时丢失（被容器的 overflow 裁剪切除）；
3. 需要按照图二 PDF 100% 还原排版样式、字体（标题 28px 黑体、正文宋体）。

## 核心修改内容

### 1. 前后端 Tiptap 样式与布局重构
- **引入高对比度 A4 纸张效果**：
  - 将前台 `AITablePanel.tsx` 和后台 `AITemplateManagement.tsx` 的编辑器外层容器背景色设为浅灰/深灰底层（`bg-[#F3F4F6] dark:bg-[#1E1F22]`）。
  - 中间编辑器本体 `.ProseMirror` 设为 `bg-white` 纯白，并设定了 `max-width: 1000px; width: 100%; margin: 16px auto` 让纸张居中加宽，并且通过 `box-shadow` 和浅灰色微弱边框模拟白纸浮起边缘。
- **显式页边距呈现**：
  - 在 `.ProseMirror` 中注入 `padding: 48px 56px !important;` 并在四周自动形成极具印刷质感的留白（页边距），确保表格与文字内容居中。

### 2. 修复表格外框线切边丢失 Bug
- **原因**：Tiptap 渲染表格时会在外面包一层 `.tableWrapper` 类容器，其自带 `overflow` 机制导致表格四周 2px 的外边框像素被隐式剪切遮挡，造成四周无边框的视觉漏洞。
- **解决方案**：为前后台的 `.tableWrapper` 样式加入强制边距避让 `padding: 4px !important;` 以及 `background-color: #ffffff !important`，使外边框得到完整展现。

### 3. 公文字体 100% 还原
- 在 `.ProseMirror` 的 CSS 规则中加入对表格单元格、表头及段落的强制字体覆盖：`font-family: 宋体, SimSun, STSong, serif !important`。
- 修改 `re_extract_tables.py` 中 `REPORT_HTML` 的大标题行，统一与 `COMPLAINT_HTML` 对齐为 `font-size: 28px; font-family: 黑体, SimHei, sans-serif; font-weight: bold; letter-spacing: 6px;`。
- 新增对 `p[align="center"]`（标题段落）的 CSS 黑体强关联规则，使得标题恒定渲染为大号粗黑体。
