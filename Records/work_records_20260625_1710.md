# 2026-06-25 17:10 全局 index.css 特异性边框冲突修复记录

## 修改背景与目的
在上一轮修复后，Master 反馈表格的外边框依然是浅灰色甚至右边框完全没有。

## 核心修改内容与排查结论

### 1. 表格浅灰色与丢失边框根因分析
*   **全局 `src/index.css` 的高特异性 Tiptap 表格样式冲突**：
    在 `src/index.css` 文件的第 162 行至 267 行中，预设了大量的 Tiptap 表格覆写样式，以实现普通现代管理系统的圆角斑马纹表格：
    1. `.ProseMirror table` 中设置了 `border: 1px solid #E5E7EB`（**造成外框依然是浅灰色**）；
    2. `.ProseMirror table td:last-child` 强制设置了 `border-right: none`（**造成表格最右侧黑线完全没有**）；
    3. `.ProseMirror table tr:last-child td` 强制设置了 `border-bottom: none`（**造成表格最下方下边线被抹除**）；
    4. 带有奇偶数斑马纹的背景色覆盖。
    由于全局 CSS 具有极为具体的 `:last-child` 选择器，其权重高过了我们在组件内部对 `.ProseMirror table td` 的覆盖，从而在渲染时吞噬了黑色的外边框。

### 2. 解决方案实现
*   **命名空间包装与超高特异性重置**：
    1.  在前台 `AITablePanel.tsx` 与后台 `AITemplateManagement.tsx` 中，为 `<EditorContent />` 外层套上一个具有专属性的包装容器：
        ```tsx
        <div className="ai-document-editor w-full">
          <EditorContent editor={editor} />
        </div>
        ```
    2.  在组件的内部 `<style>` 标签中，利用 `.ai-document-editor` 命名空间前缀，通过 `!important` 强力且彻底地将 index.css 中的干扰项覆写归零：
        *   将 `border-collapse` 设为 `collapse`，圆角设为 `0 !important`，阴影清空；
        *   重置最右侧列单元格的右边框为黑色实线：
            ```css
            .ai-document-editor .ProseMirror table td:last-child,
            .ai-document-editor .ProseMirror table th:last-child {
              border-right: 1px solid #000000 !important;
            }
            ```
        *   重置最后一行的下底框为黑色实线：
            ```css
            .ai-document-editor .ProseMirror table tr:last-child td {
              border-bottom: 1px solid #000000 !important;
            }
            ```
        *   清除所有奇偶斑马纹与 hover 时变灰/变蓝的情况，全部锁定纯白底：
            ```css
            background-color: #ffffff !important;
            background: #ffffff !important;
            ```
        *   将表格最外圈四周边框强力锁定为 `border: 2px solid #000000 !important;`。
