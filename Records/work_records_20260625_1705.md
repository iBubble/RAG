# 2026-06-25 17:05 表格全黑边框与上下缺失框线修复记录

## 修改背景与目的
在上一版本将白色 A4 纸张、页边距与宋体还原后，Master 反馈发现表格的最上方和最下方的外框线依然不见了（呈现白色虚空状，完全缺失）。

## 核心修改内容与排查结论

### 1. 表格缺失框线根因分析
*   **Tailwind Typography 强力重置干扰**：
    在编辑器引入的 `.prose` 类库中，默认会对 `table` 子组件应用预设。其中有两条极具破坏力的样式：
    1. `.prose tbody tr:last-child` 会被强行设为 `border-bottom-width: 0`；
    2. `.prose tr` 本身没有顶边框，只有底边框。
    这导致在 `border-collapse: collapse` 作用下：
    *   表格第一行的顶端没有边框支持，导致**上方黑线彻底缺失**；
    *   最后一行的底端被强制覆盖为宽度 0，导致**下方黑线彻底缺失**；
    *   表格的外围 `table` 边框在折叠竞争中被 `prose` 的 `border: none` 抹除。

### 2. 解决方案实现
*   **多向全黑色边框显式约束**：
    在前台 `AITablePanel.tsx` 与后台 `AITemplateManagement.tsx` 的全局 CSS 中，针对单元格进行了四向硬编码式黑色框线绑定：
    ```css
    .ProseMirror table td, .ProseMirror table th {
      border: 1px solid #000000 !important;
      border-top: 1px solid #000000 !important;
      border-bottom: 1px solid #000000 !important;
      border-left: 1px solid #000000 !important;
      border-right: 1px solid #000000 !important;
    }
    ```
*   **清除行级边框重置**：
    在 CSS 中重置行级容器的默认边框机制：
    ```css
    .ProseMirror tr, .ProseMirror tbody tr, .ProseMirror tr:last-child, .ProseMirror tbody tr:last-child {
      border: none !important;
      border-top: none !important;
      border-bottom: none !important;
    }
    ```
    通过把行的边框限制全权交还给单元格四向边框，避开了 Tailwind Typography 的边框截断覆盖，确保了四周外框线和内网格线均以黑色实体线完整呈现。
