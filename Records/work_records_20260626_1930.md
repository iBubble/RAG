# 工作修改记录 - 2026-06-26 19:30

本次修改主要完成了“优化方案 B”下的 RAG 检索管道去噪参数微调，以及前端侧边栏的 UI 简化和一键引用系统中所有公共文档库的重构。

## 修改内容说明

### 1. 前端部分
*   **[TreeView.tsx](file:///Users/gemini/Projects/Own/RAG/app-server/frontend/src/components/TreeView/TreeView.tsx)**：
    *   移外部移除了 `activeTab` 状态，彻底隐藏并去掉了“本案文档”和“公共文档”分类切换 Tab，只保留本案文档的单文件树展示。
    *   新增了 `usePublicRef` 状态（默认为 `true`），并在本案文档目录树上方添加了一个莫兰迪色调的 `[✔] 引用所有公共文档` 复选按钮。
    *   新增 `useEffect` 联动机制：当启用“引用所有公共文档”时，自动将所有公共文档 ID 追加至 `checkedRefIds` 中；当取消勾选时，清空该列表，以此配合后端实现完全纯净的案情问答或双路法条关联。
    *   清理了不再使用的 `refLoading` 状态、`seenRefIdsRef` 引用以及未使用的 `X` 图标导入，确保 TypeScript 编译无 unused locals 报错。

### 2. 后端部分
*   **[projects.py](file:///Users/gemini/Projects/Own/RAG/app-server/backend/api/projects.py)**：
    *   重构了 `/{case_id}/ref-files` 接口。从原本只拉取“当前案件关联引用的项目文件”重构为**动态获取系统中所有公开的 `project_type='library'` 的公共文档库中的所有文件**。
    *   摆脱了需要逐个项目手动做引用配置的复杂流程，从而在启用“引用所有公共文档”时，能够实时、动态全选系统内所有的公共文档（在当前数据库环境下为 561 个公共文档）。
*   **[intent_classifier.py](file:///Users/gemini/Projects/Own/RAG/app-server/backend/core/intent_classifier.py)**：
    *   在策略模板 `_STRATEGY_TEMPLATES` 中，将 `comparison`（对比）和 `general_qa`（常规问答）的 `vector_top_k` 参数从原本的 `12` 降低至最优去噪阈值 `8`，使检索上下文切片数量在全局受限在最精细的 `6 ~ 8` 个切片内。
*   **[retrieval_pipeline.py](file:///Users/gemini/Projects/Own/RAG/app-server/backend/core/retrieval_pipeline.py)**：
    *   将动态获取策略 fallback 时的 `vector_top_k` 默认值从 `12` 调整为 `8`。
*   **[generate.py](file:///Users/gemini/Projects/Own/RAG/app-server/backend/api/generate.py)**：
    *   将缓存指纹 `_strategy_fp` 生成时策略中 `vector_top_k` 的兜底 fallback 值由 `12` 调整为 `8`。
