## 📝 具体修改内容与设计决策
由于 Chrome 浏览器原生 `window.confirm` 在 React 重绘、焦点争夺或事件穿透时极易一闪而过、默认返回 `false`，我们执行了以下改造：
1. **剥离原生 Confirm 判断**：将上述三个模块的 `window.confirm` 询问全部移除，改为使用前端 React 状态及高颜值 Modal 组件进行接管。
2. **自定义 Portal Modal 重构**：在 UI 组件层，采用 `createPortal` 将美观度更高的 `ConfirmModal` 直接挂载至 `document.body`，规避了因 DOM 祖先节点样式导致的交互及焦点冲突。
3. **防冒泡及默认行为拦截**：为所有的触发按钮 onClick 事件加装了 `e.preventDefault()` 及 `e.stopPropagation()`，彻底杜绝冒泡副作用。
4. **大纲判断与 Hook 解耦**：将 `useGenerationQueue.ts` 中的 confirm 直接剥离，由 `DocumentStudio.tsx` 外层控制弹窗的显示，使 Hook 职责更加纯粹。
