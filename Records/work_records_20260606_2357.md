本次调整主要重构了后台“学习看板”（LearningProgress）的案件排序逻辑，实现了按优先级和创建时间双重维度的智能排序。

## 2. 修改细节说明

### A. 后端获取数据字段扩展（admin.py）
- 在 `get_learning_progress` 的数据库查询中扩展了 `created_at` 字段的提取：
  `projects = conn.execute("SELECT id, name, priority, is_paused, created_at FROM projects").fetchall()`
- 在 API 的返回体 JSON 中，新增了 `"createdAt"` 属性的绑定：
  `"createdAt": dict(p).get("created_at", ""),`

### B. 前端界面展示与排序逻辑重构（LearningProgress.tsx）
- 扩展了前端 `ProjectProgress` 接口的 TypeScript 定义，支持 `createdAt?: string` 属性声明，彻底解决了 tsc 的静态类型检查与构建报错。
- 重构了前端 `sortedProjects` 记忆排序块的 `useMemo` 回调：
  - **优先级排序**：数值从小到大（ASC，1 级优先级最先展示，接着是 2 级、3 级）。
  - **创建时间降序排序**：对于优先级相同的案件，通过 `dateB.localeCompare(dateA)` 执行字母序逆序比对，实现了“最新创建的案件最靠前展示”的规则。

### C. 前端编译与服务重启
- 执行了 `npm run build` 对修改后的前端进行打包编译。
- 执行了 `pm2 reload all` 对容器内所有服务进程（FastAPI 后端、Celery Workers、Vite preview 静态前端）执行了平滑重载。
