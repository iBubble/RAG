## 1. 修改文件与具体内容

### ① `docker-compose.yml`
*   **修改内容**：为 `langfuse` 容器服务补充注入了 `ENCRYPTION_KEY` 与 `SALT` 环境变量。
*   **原因**：修复自托管 Langfuse 启动时安全校验失败导致的 `500 Internal Server Error` 问题。

### ② `app-server/backend/core/audit_trace.py`
*   **修改内容**：在 `write_audit_trace` 函数中新增 `trace_id` 参数输入支持；将底层 SQLite 执行逻辑中的 `INSERT INTO` 改为 `INSERT OR REPLACE INTO`。
*   **原因**：实现以相同的 `trace_id` 在拦截挂起和 Resume 恢复之后能原地覆盖更新审计记录。

### ③ `app-server/backend/api/generate.py`
*   **修改内容**：
    1. 底部新增 `POST /api/internal/trace/audit` 路由，用于供 Go 网关向 Python 后端上报最终审计状态。
    2. 在常规 `chat` 接口流式生成完毕后，增加 `write_audit_trace` 自动调用，确保非 smart 模式下的回答也能落库并推送。

### ④ `app-server/nexus-gateway/eino_graph.go`
*   **修改内容**：
    1. 在 `EinoContext` 结构体中添加 `TraceID` 字段；引入 `"github.com/google/uuid"` 并在 `RunEinoOrchestration` 入口处动态生成 `TraceID`。
    2. 在 `plannerNode`、`workerNode`、`checkerNode` 和 `auditorNode` 的 Lambda 起止处添加高精度计时测算，并向 Python 发送 `/api/internal/trace/span` 建立子 Span。
    3. 在 `checkerNode` 触发挂起以及 `auditorNode` 正常结束时，调用新增的 `reportAuditTraceToPython` 将记录推入审计表。
    4. 调整 `freezeEinoState` 与 `RunEinoResumeOrchestration` 以在拦截挂起/ Resume 恢复时携带并恢复相同的 `trace_id` 状态。

### ⑤ `app-server/nexus-gateway/chat_handler.go`
*   **修改内容**：在 `ResumeHandler` 人工恢复处理中提取 Redis 冻结状态里的 `trace_id`，并将其顺延传递至 `RunEinoResumeOrchestration` 恢复流中。
