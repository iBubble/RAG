五、通用工作规范（方法论，无需改动）

引号规范
全部使用全角引号（中文引号），禁止使用半角引号。适用于所有输出物：批注、法律意见书、对照表等。

批注生成技术规范
使用 Python + lxml 直接操作 docx（zip 包）实现：
1. 创建 word/comments.xml（批注内容）
2. 在 word/document.xml 对应段落插入 commentRangeStart、commentRangeEnd、commentReference
3. 在 [Content_Types].xml 添加 comments 的 ContentType
4. 在 word/_rels/document.xml.rels 添加 comments 的 Relationship
5. 生成前验证：所有批注文本中不含半角引号

批注时间生成规则
- 用 datetime.now() 获取当前真实时间，禁止硬编码时间。
- 第一条批注时间 = 当前系统时间往前随机 1-3 分钟。
- 时间方向递增，从第二条起每条在前一条基础上加 1 或 2 分钟。
- 最后一条不得超过当前系统时间（不出现未来时间）；批注多导致超时则缩短间隔或起始时间进一步前移。
- 格式：YYYY-MM-DDTHH:MM:00Z

批注格式要求
- 批注人：见上方"律师信息"中的批注署名。
- 批注形式：直接在 Word 文档中生成批注（Comment），不是文本修改。
- 批注标签：每条以【修改建议】【审查意见】【重点审查意见】开头。
- 批注正文禁止出现"A档/B档/C档/小额/中等/大额"等档位字样；档位仅用于内部判断审查深度。生成前校验。

批注层级定义
【修改建议】——文字表述问题（语句不通、用词不当、引号错误、笔误）
【审查意见】——一般性法律完善建议（补充约定、明确标准、潜在风险提示）
【重点审查意见】——重大风险或需客户决策事项（权利义务严重失衡、核心条款缺失、合规红线）

六、法律意见书撰写规范

行文结构（备忘录形式）：
备忘录 / 保密 / 致：[按客户档案主体名称] / 自：[律所全称] / 日期 / 关于：[按客户档案命名规则] / 敬启者： / 一、基本情况 / 二、相关法律规定 / 三、法律分析 / 四、法律意见 / 结尾礼语 / 律所落款 + 日期。

各部分要求：
1. 开头段落：说明受委托事项、分析前提（如"基于贵司提供的材料及说明的事实"）、分析边界与定位；与此前意见书有关联的在此说明。
2. 基本情况：按法律关系链条组织事实，每段为后面分析铺设逻辑基础；区分书面依据事实与口述事实。
3. 相关法律规定：只列分析中真正用到的法条，每条都能在分析中找到对应论证；标注法律全称和制定/修订年份；禁止为凑篇幅堆砌。
4. 法律分析（核心）：结论先行；不重复抄法条原文，只提名称和条文编号；从法条提炼规则要点对接具体事实完成"规则→事实→结论"推演；顾问口吻；先讲结构性核心问题再讲具体合规事项。
5. 法律意见：给明确结论和可操作建议，具体到可执行步骤，不重复法律分析内容。

格式要求（便于粘贴 Word）：最终意见书 md 文件内容为纯文本风格——禁止加粗、斜体、井号标题、分割线、markdown 表格、短横线列表；分层用中文序号体系（一、二、三；（一）（二）（三）；1. 2. 3.；（1）（2）（3））；强调靠行文本身；全角引号；粘贴 Word 后无需清理符号。过程文件不受此限制。

写作禁忌：禁止合规清单式写法；禁止法条堆砌；禁止分析与意见重复；禁止空洞风险提示（必须说清风险是什么、后果是什么、怎么应对）。

七、法律法规检索要求
- 所有引用的法律法规必须准确、现行有效、版本最新；已修改或废止的不得引用。
- 引用法条标注法律全称及制定/修订年份；地方文件标注文号和有效期。
- 法条引用服务于分析论证，不得大段堆砌原文；某层级确无相关规定的不引用、不提及，不为凑层级硬加。
- 禁止凭记忆引用法条。所有法条必须经在线检索验证（国家法律法规数据库、北大法宝、政府官网等），不得直接从训练数据调取。
- 无法验证的法条标注【待查】，不得默认正确直接写入。
