合同审查引擎

本文件为合同审查的核心流程引擎。使用前须已读取律师档案和客户档案。所有路径、署名、分级标准、立场均来自档案，本引擎不写死任何具体客户或律师信息。

一、审查前准备

1. 从客户档案获取：客户简称、主体信息、立场声明、客户类型、分级标准、工作目录配置、特殊审查要点；如客户文件夹内放有企查查/企业信用报告，一并读取作为对方主体核查依据。
2. 从律师档案获取：批注署名、批注技术规范、批注时间生成规则、引号规范、批注标签定义。

二、执行步骤

步骤1：读取合同文件
从客户档案指定的输入目录（或用户指定路径）读取 .docx 文件。

步骤2：识别签约主体
- 确认我方在合同中的角色（甲方/乙方/其他）。
- 如客户有多个关联主体，确认是哪个主体签约，匹配对应信息和立场。

步骤3：判定审查档位
按客户档案中的分级标准和分级判定规则确定 A/B/C 档。档位仅用于内部判断审查深度和批注密度，不写入批注正文。

步骤4：加载审查清单
先加载通用审查要点。再按客户类型加载对应的一个专项清单：

  国有控股企业            → 国企专项
  民营企业（普通有限公司）→ 民营企业专项
  民营初创企业            → 民营初创专项
  外资企业                → 外资专项

如合同判定为 C 档，或合同中涉及保证、抵押、质押、定金等担保条款，额外加载担保专项。
审查时将清单中的"我方"替换为客户档案中的客户简称。

步骤5：逐项审查
按已加载清单中适用于当前档位的要点逐一检查。把握两条原则：
- 缺失条款检查项必须逐项核对（模型看不见不存在的条款，最易漏）。
- 已有条款的质量判断（违约责任是否对等、价格是否明确、违约金是否畸高等）依靠自身专业能力把握，不必拘泥清单是否列出。
侧重点由档位和批注密度上限控制，避免面面俱到、淹没重点。批注密度参考客户档案分级标准。

步骤6：生成批注版合同
按律师档案中的批注技术规范，使用 Python + lxml 在 docx 中插入批注（Comment，非文本修改）。
- 批注署名、批注时间按律师档案规则生成。
- 引号全部使用全角引号，生成前校验不含半角引号。
- 每条批注以【修改建议】【审查意见】【重点审查意见】开头；批注正文不得出现"A档/B档/C档/小额/中等/大额"等档位字样，生成前校验。
- 输出到客户档案指定的输出目录，文件命名按客户档案配置。

步骤7：终端输出审查摘要
在终端输出审查要点摘要，包含：
- 合同金额与审查档位（仅终端显示，不写入批注）
- 重大风险提示（条款 + 风险等级 + 核心问题）
- 重大条款缺失清单
- 各条款审查意见汇总

步骤8：询问是否生成修改建议对照表（可选）
询问用户是否需要额外生成修改建议对照表（markdown 表格）。如需要，按以下格式生成并保存为 修改建议对照表-原文件名.md，存放于输出目录：

  序号 | 条款位置 | 原文摘要 | 问题 | 修改建议 | 风险等级 | 法律依据

步骤9：更新台账（可选，仅当客户档案启用台账管理时执行）
按客户档案中的台账列定义，使用 Python + openpyxl 更新合同全流程管理台账。先读取台账确认当前最大序号和下一空行，避免覆盖已有数据，保持与已有行格式一致。

三、各档审查策略差异

A档——底线把控
重点关注：主体资质、价格明确性、基本违约责任、管辖法院。适当放宽：缺失的送达、保密、反商业贿赂等附属条款合并为一条综合提醒；违约金比例不苛求精确对等；权利义务对等性做整体判断。以【审查意见】为主，重大问题才用【重点审查意见】。

B档——标准审查
按已加载清单完整覆盖，各项按默认标准执行。三类标签均可。

C档——深度审查
在 B 档基础上额外强化：履约担保（缺少标【重点审查意见】，并按担保专项深度审查）、付款节奏（逐笔核对付款节点与交付里程碑，预付款建议不超过30%）、退出机制（细化提前终止触发条件、善后、成本分担）、对方履约能力（结合企查查报告核查注册资本与合同金额匹配度）、争议解决升级（评估仲裁、诉前财产保全）。重大失衡、核心缺失、需客户决策的一律【重点审查意见】。
