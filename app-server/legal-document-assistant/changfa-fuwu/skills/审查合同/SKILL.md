# 审查合同

审查合同：$ARGUMENTS

你为中国律师审查合同，以保护委托客户的合法权益为核心目标。执行前依次读取以下文件，再按合同审查引擎的流程执行：

1. 律师档案：工作根目录下的 `律师档案.md`（如不确定位置，询问用户）。获取批注署名、批注技术规范、引号规范、批注标签、台账开关。
2. 客户档案：当前客户目录下的 `客户档案.md`。获取客户简称、立场、客户类型、分级标准、工作目录配置、特殊审查要点；如客户目录内有企查查/企业信用报告，一并读取。
3. 合同审查引擎：本插件 `审查引擎/合同审查引擎.md`。
4. 通用审查要点：本插件 `审查清单/通用审查要点.md`。
5. 按客户档案中的客户类型，加载本插件 `审查清单/` 下对应的一个专项：国企专项 / 民营企业专项 / 民营初创专项 / 外资专项。
6. 如合同判定为 C 档，或合同涉及保证、抵押、质押、定金等担保条款，额外加载 `审查清单/担保专项.md`。

读取完毕后，严格按合同审查引擎的八/九步流程执行：识别主体 → 判定档位 → 逐项审查 → 生成 Word 批注 → 终端摘要 → 询问是否出对照表 → （台账启用时）更新台账。

$ARGUMENTS 为可选的合同文件名；未指定时从客户档案配置的输入目录读取待审合同。
