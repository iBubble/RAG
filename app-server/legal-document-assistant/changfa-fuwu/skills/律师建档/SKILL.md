# 律师建档（一次性冷启动）

你在帮一位中国律师完成常法服务套件的首次配置。目标是生成一份"律师档案"，供套件内所有技能共享。全程中文、口吻简洁，一次问清，不要让用户反复填。

## 工作流程

### 第一步：读取模板
读取本插件根目录下的 `律师档案-模板.md`，了解需要填写的字段和保留的方法论部分。方法论部分（引号规范、批注技术规范、批注时间规则、批注层级、法律意见书撰写规范、检索要求）原样保留，不要改动，也不要问用户。

### 第二步：收集律师信息
依次询问（可一次性提供）：
1. 律师姓名
2. 律所全称
3. 批注署名（Word 批注里显示的署名，可用律所简称或个人简称）
4. 工作根目录（你打算存放各客户文件夹的目录，如 `~/Desktop/常法客户/`；不确定就用这个默认值）

### 第三步：勾选启用的技能
问用户这次想启用哪些（多选）：合同审查、项目支持、委托续约检查、审查复盘。新客户建档默认保留。
未勾选的技能不写入"启用"，但不影响其他技能运行，日后可随时改律师档案启用。

### 第四步：确认台账开关
告知用户台账管理默认启用（合同审查和项目支持完成后自动登记台账）。问一句是否保留。
- 保留 → 台账管理：启用
- 不需要 → 台账管理：关闭（相关登记步骤自动跳过）

### 第五步：确认默认分级标准
展示模板里的默认分级标准（A≤10万 / B 10-100万 / C＞100万）。说明：这是默认值，安装后可随时调整，新客户建档时还会按客户注册资本给出针对性推荐。问用户现在是否需要调整，不需要就用默认。

### 第六步：生成律师档案
把以上信息填入模板，生成 `律师档案.md`，保存到用户指定的工作根目录下。方法论部分原样保留。
生成后向用户复述：律所/律师/署名、工作根目录、已启用技能、台账开关、分级标准。

### 第七步：引导下一步
提示用户：律师档案已就绪。接下来对每一个常法客户运行一次"新客户建档"，为该客户建立客户档案和工作目录，即可开始审合同、做项目支持。

## 注意
- 这是一次性配置，不是每次审查都要跑。
- 律师档案随时可手动编辑，改完立即生效。
- 不收集任何客户信息——客户信息属于"新客户建档"。
