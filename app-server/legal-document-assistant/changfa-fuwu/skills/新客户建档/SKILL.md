# 新客户建档（每客户一次）

你在帮中国律师为一个新的常法客户建档。运行前应已有"律师档案"（由律师建档生成）。全程中文。

## 工作流程

### 第一步：读取律师档案
从工作根目录读取 `律师档案.md`，获取工作根目录、台账开关、默认分级标准、律所信息。如未找到律师档案，提示用户先运行"律师建档"。

### 第二步：收集客户主体信息
优先方式：提示用户可以把该客户的企查查报告或企业信用报告放进客户文件夹（或直接粘贴文字/截图），你来读取并自动提取主体信息。
如未提供或信息不全，再就缺失项逐项追问。需要的信息：
必填：客户简称、全称、统一社会信用代码、企业类型、注册资本/实缴、成立日期、法定代表人、注册地址、经营范围（核心）、客户类型。
选填：财务负责人、纳税人资质、开户行及账号、股东结构、实际控制人、关联企业、登记机关。
客户类型四选一：国有控股企业 / 民营企业（普通有限责任公司）/ 民营初创企业 / 外资企业。

### 第三步：确认配置（系统推断，用户确认）
1. 适用专项清单（按客户类型自动对应）：
   - 国有控股企业 → 通用审查要点 + 国企专项
   - 民营企业（普通有限责任公司）→ 通用审查要点 + 民营企业专项
   - 民营初创企业 → 通用审查要点 + 民营初创专项
   - 外资企业 → 通用审查要点 + 外资专项
   （担保专项在合同判 C 档或涉担保条款时自动叠加，无需在此选。）
2. 合同审查分级标准：按注册资本推荐，用户可调整。
   - 大型（注册资本≥5000万）：A≤10万 / B 10-100万 / C＞100万
   - 中型（500万-5000万）：A≤5万 / B 5-50万 / C＞50万
   - 小型（＜500万）：A≤3万 / B 3-30万 / C＞30万
3. 管辖法院目标：默认客户所在地人民法院，可调整。
4. 意见书命名前缀、致送对象：默认用客户简称/全称，可调整。
5. 工作目录编号：在工作根目录下顺延编号。

### 第四步：生成客户档案
在 `工作根目录/<编号>-<客户简称>/` 下生成 `客户档案.md`，包含：
客户属性（简称、客户类型、适用专项清单、意见书命名前缀、致送对象）、立场声明（始终站在该客户一方）、主体信息表、合同审查分级标准及判定规则、工作目录配置、客户特殊审查要点（如有，如国企的体制/检索层级要求）。
如律师档案台账开关为"启用"，在客户档案中写入合同审查台账和项目支持台账的列定义。

### 第五步：创建目录结构
在客户文件夹下创建标准结构（用 Bash mkdir）：
```
<编号>-<客户简称>/
├── 客户档案.md
├── 0-委托合同/
├── 1-项目支持/
│   ├── 过程文件/
│   └── 出具意见书/
└── 2-合同审查/
    ├── 合同原版/
    └── 律师修改版/
```
如台账启用，在对应目录创建空白台账文件（用 openpyxl 按客户档案的列定义建表头）。

### 第六步：完成提示
输出摘要：客户档案位置、适用专项清单、分级标准、工作目录结构。
提示用户：
- 把待审合同放入 `2-合同审查/合同原版/`，运行合同审查即可。
- 如已放入企查查报告但仍有信息缺口，告知缺哪些、请补充。
- 如启用台账，提醒把委托合同信息补充到委托合同台账（供委托续约检查使用）。

## 注意
- 客户类型决定加载哪个专项，务必确认准确。
- 不臆造主体信息：企查查报告没有、用户也未提供的字段，标注待补充，不编造统一社会信用代码、注册资本等。
- 立场声明必须明确写入：无论该客户在合同中是甲方乙方，均以保护其合法权益为核心。
