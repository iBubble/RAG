# 常法服务（changfa-fuwu）

服务一个常年法律顾问客户全生命周期所需的技能集。基于中国大陆法律体系，面向中国执业律师的常法日常工作。

技能本身是工具无关的 markdown 指令文件（SKILL.md、引擎、清单、档案），在 Claude Code 里可作为插件一键安装，技能内容本身也可供 Codex 等 agent CLI 使用。

核心设计：引擎与客户信息分离。改审查逻辑，改一次所有客户生效；加新客户，建一份客户档案即可开工。

## 它能做什么

- 合同审查：按 A/B/C 金额分级审查，自动在 Word 文档生成批注，输出审查摘要；按客户类型加载对应专项清单。
- 项目支持：以战略法律顾问视角分析项目，产出咨询回复或法律意见书；可选案例检索（北大法宝）。
- 委托续约检查：扫描委托合同到期情况，按紧急程度分级提醒。
- 审查复盘：统计某客户合同的高频问题，生成合同模板优化建议。
- 律师建档 / 新客户建档：两个冷启动，分别配置你自己和每一个客户。

## 上手三步

1. 运行一次「律师建档」。填写律所/律师/批注署名、工作根目录，勾选要启用的技能，确认台账开关和默认分级标准。生成你的「律师档案」，所有技能共享。
2. 对每个常法客户运行一次「新客户建档」。可把该客户的企查查报告放进文件夹由系统提取主体信息，按客户类型确定专项清单、按注册资本推荐分级，生成「客户档案」和工作目录。
3. 开始日常工作：把合同放进客户的 `合同原版/` 目录运行「审查合同」；需要项目分析时运行「项目支持」。

## 目录结构

```
changfa-fuwu/
├── skills/                 业务技能与冷启动（六个）
│   ├── 律师建档/           一次性冷启动
│   ├── 新客户建档/         每客户冷启动
│   ├── 审查合同/
│   ├── 项目支持/
│   ├── 委托续约检查/
│   └── 审查复盘/
├── 审查引擎/               合同审查引擎、项目支持引擎
├── 审查清单/               通用 + 国企/民营企业/民营初创/外资/担保
```

## 三层配置

- 律师级：律师档案（你的署名、规范、默认分级、技能与台账开关），所有客户共享，建一次。
- 客户级：客户档案（主体信息、立场、客户类型、分级、特殊要点），每客户一份。
- 任务级：每次审查/每个项目的具体处理。

## 审查清单的设计原则

清单只装"模型不会主动想到、或需要统一口径"的内容，不重复罗列模型本身已掌握的通用法律常识。
经 A/B 对照测试验证：对已有条款的质量判断（违约责任是否对等、价格是否明确等），模型看着原文即可挑出，不必写进清单；对是否缺失某条款（解除权、送达、保密等），模型看不见不存在的文字，不提示就会漏——故清单砍掉质量判断类，保留全部缺失检查类。侧重点由 A/B/C 分级和批注密度上限控制。

## 重要纪律

- 所有输出均为供律师审阅的草稿，最终判断由执业律师负责。
- 法条必须经在线检索验证，禁止凭记忆引用，无法验证的标【待查】。
- 不臆造主体信息：缺失字段标注待补充，不编造。
- 向客户发出的文件为纯文本风格，不使用 markdown 符号，便于直接粘贴 Word。
