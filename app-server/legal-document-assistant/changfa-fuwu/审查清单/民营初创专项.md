民营初创企业合同专项审查清单

加载条件：客户档案中 客户类型=民营初创企业 时自动加载。
使用方式：通用审查要点的叠加模块，在完成通用审查的基础上额外审查以下专项要点。
设计原则：只列初创企业特有的增量要点（主体形态、印章不规范、行业许可、场地依赖）；通用质量判断（如对方是否利用优势压条款）交由通用清单和模型自身能力处理，不重复。

一、签约主体形态（初创常以个体户/新设主体签约，模型不会主动想到）
- [ ] 以个体工商户名义签约的：主体表述用全称，经营者处填经营者姓名 [全档]
- [ ] 个体户经营者以个人财产承担无限责任，风险敞口较大时建议以公司名义签约 [全档]
- [ ] 涉及特定许可经营项目（餐饮、酒类等）的，应以持有相应许可的主体名义签约 [全档]

二、主体混同风险（初创多主体、印章不规范高发）
- [ ] 是否存在关联主体混用（如抬头写公司却盖个体章）[全档]
- [ ] 多主体共用同一场所经营的，注意区分各主体的独立性 [全档]

三、初创特有缺失检查
- [ ] 合同含"经营业绩""历史数据"等表述的，检查是否缺少数据真实性承诺/保证条款 [全档]
- [ ] 涉及餐饮、食品销售的，检查是否缺少食品经营许可证要求、食品安全责任条款 [全档]
- [ ] 经营场所集中于单一商业综合体的，关注租赁稳定性、续租条件、转租限制对合同履行的影响 [B/C档]
